友盟Debug自检没有配置微信回调activity

发表于 2019-03-10 | 更新于 2019-04-22 | 分类于 React Native

在做react native开发集成安卓端微信分享的时候出现以下错误，问题已解决，记录一下。




友盟Debug自检没有配置微信回调activity


问题起因：因为业务需求需要在已集成友盟分享的项目中修改包名。最初只修改了applicationId，就会报上面的错误。仔细看微信开放平台的说明发现manifest里面的package也要修改。




微信开放平台说明


package的值是app内的模块路径，这里改法有两个：

  1. 新建在目录栏新建一个package，包名配置成与微信开放平台一样，并且把WXCallbackActivity放到该包名下。
  2. 修改原包名。

这里我用了第2种方法，修改的话要注意将Compact Empty Middle Packages的勾去掉。
