事件分发机制

事件分发机制大体可以分为三部分 事件生产 事件分发 事件消费

如果所有组件都不消费该事件，那么最终用户的行为就不会有任何响应

安卓没有iOS的hitTest，因此在一个事件序列中所有事件传递和响应方法都会触发多次

事件传递

事件传递顺序：Activity -> Window(实现类:PhoneWindow)->decor view(底层容器) -> View

安卓的Window在Activity之下，View之上，与iOS不同

Window用于接收触摸事件，并传递给Activity

boolean dispatchTouchEvent(MotionEvent ev)

  1. 返回false：不再向下分发且从该View的上一级的onTouchEvent事件向上传递（不会执行自己的onTouchEvent）

  2. 返回true：表示事件直接被消费，这个事件也就停止分发且不会逆向向上传递，直接结束了。

  3. 返回super：事件将会继续向下分发，直到事件被消费为止。

boolean onInterceptTouchEvent(MotionEvent ev)

view没有这个方法，只有ViewGroup有

  1. 返回false/super（默认）:不做此次拦截，事件将会正常向下分发，分发至下级的dispatchTouchEvent方法 再次判断是否分发事件。

  2. 返回true:表示ViewGroup容器拦截后续事件，会执行该ViewGroup的onTouchEvent（）方法然后停止向下分发转而通过onTouchEvent（）向上传递，直到最终被消费

事件响应

响应顺序：监听器—>view组件的回调方法—>Activity的回调方法

响应事件可通过两种方式，重写view的响应方法或给view添加事件监听器，其中响应方法返回值

  • 返回true：立即消费掉事件，事件将不会向上传递，事件到此终止。

  • 返回false/super：不消费掉此次事件，事件将会层层向上传递，直到被消费。

OnTouchListener相关方法与属性:

event.getAction == MotionEvent.ACTION_DOWN：按下事件
event.getAction == MotionEvent.ACTION_MOVE：移动事件
event.getAction == MotionEvent.ACTION_UP：弹起事件

event.getAction == MotionEvent.ACTION_POINTER_DOWN:当屏幕上已经有一个点被按住，此时再按下其他点时触发

event.getAction == MotionEvent.ACTION_POINTER_UP:当屏幕上有多个点被按住，松开其中一个点时触发（即非最后一个点被放开时）

getX/getY返回的是相对于当前View左上角的x和y坐标，而getRawX/getRawY返回的是相对于手机屏幕左上角的x和y坐标。

EventBus

跟iOS的Notification差不多

在 EventBus 3.0 之前消息处理的方法只能限定于 onEvent、onEventMainThread、onEventBackgroundThread和onEventAsync，它们分别代表4种线程模型。

threadMode有五种类型（POSTING、MAIN、MAIN_ORDERED、BACKGROUND、AYSNC）

  • POSTING：接收者与发送者同线程

  • MAIN：接收者在主线程并同步阻塞

  • MAIN_ORDERED：接收者异步回到主线程

  • BACKGROUND：如果发布在子线程那么直接在发布线程调用，如果发布在主线程那么将开启一个子线程来调用

  • AYSNC：在子线程调用，总是开启一个新的线程来调用

sticky的作用是在先post事件，在事件发送之后注册的订阅者也能收到，通过removeStickyEvent移除粘性事件。默认false

priority参数是int型，默认值是0，优先级

drawable

安卓中有13种drawable，用于渲染控件的样式，Drawable常被用来作为View的背景使用

  • ColorDrawable

    使用：@color/xxx；java使用：

  • NinePatchDrawable(点九图)

  • ShapeDrawable

    边框圆角等Drawable，根节点<shape>，子节点<size>,<solid>,<gradient>,<stroke>,<conner>,<padding>

    <**shape**>:

    • ~ visible:设置是否可见
    • ~ shape:形状,可选:rectangle(矩形,包括正方形),oval(椭圆,包括圆),line(线段),ring(环形)
    • ~ innerRadiusRatio:当shape为ring才有效,表示环内半径所占半径的比率,如果设置了innerRadius, 他会被忽略
    • ~ innerRadius:当shape为ring才有效,表示环的内半径的尺寸
    • ~ thicknessRatio:当shape为ring才有效,表环厚度占半径的比率
    • ~ thickness:当shape为ring才有效,表示环的厚度,即外半径与内半径的差
    • ~ useLevel:当shape为ring才有效,表示是否允许根据level来显示环的一部分

    <**size**>:

    • ~ width:图形形状宽度
    • ~ height:图形形状高度

    <**solid**>

    • ~ color:背景填充色,设置solid后会覆盖gradient设置的所有效果!!!!!!

    <**stroke**>

    • ~ width:边框的宽度
    • ~ color:边框的颜色
    • ~ dashWidth:边框虚线段的长度
    • ~ dashGap:边框的虚线段的间距

    <**conner**>

    • ~ radius:圆角半径,适用于上下左右四个角
    • ~ topLeftRadius,topRightRadius,BottomLeftRadius,tBottomRightRadius: 依次是左上,右上,左下,右下的圆角值

    <**padding**>

    • left,top,right,bottm:依次是左上右下方向上的边距!

  • GradientDrawable

    渐变Drawable，节点：<gradient>

    • android:angle（仅适用于线性渐变）：指定线性渐变的角度，以度为单位。例如，0表示从左到右的渐变，90表示从上到下的渐变，45表示从左上到右下的渐变。

  • BitmapDrawable

    位图Drawable，节点：<bitmap>

  • InsetDrawable

    表示把一个Drawable嵌入到另外一个Drawable的内部，并且在内部留一些间距, 类似margin，节点：<inset>

    • insetLeft,insetRight,insetTop,insetBottm:设置左右上下的边距

  • ClipDrawable

    裁剪Drawable，支持水平和竖直2个方向(clipOrietntion)，从哪个位置开始(gravity)，裁剪一定比例(Level，0-10000)，节点：<clip>

  • RotateDrawable

    旋转Drawable，节点：<rotate>

  • AnimationDrawable

    帧动画Drawable，调用star()开始播放，oneshot:设置是否循环播放,false为循环播放!!! duration:帧间隔时间，根节点：<animation-list>，子节点：<item>

  • LayerDrawable

    层次化图形对象，包含一个Drawable数组，然后按照数组对应的顺序来绘制他们，通过将不同的Drawable放置在不同的层上面从而达到一种叠加后的效果，索引 值最大的Drawable会被绘制在最上层，节点：<layer-list>

  • TransitionDrawable

    转场Drawable，管理两层的Drawable，根节点：<transition>，startTransition开始动画，reverseTransition反向播放

  • LevelListDrawable

    用来管理一组Drawable的，跟LayerDrawable类似，我们可以为里面的drawable设置不同的level，当他们绘制的时候，会根据level属性值获取对应的drawable绘制到画布上，根节点：<level-list>

  • StateListDrawable

    状态Drawable，根节点：<selctor>

    • drawable:引用的Drawable位图,我们可以把他放到最前面,就表示组件的正常状态~
    • state_focused:是否获得焦点
    • state_window_focused:是否获得窗口焦点
    • state_enabled:控件是否可用
    • state_checkable:控件可否被勾选,eg:checkbox
    • state_checked:控件是否被勾选
    • state_selected:控件是否被选择,针对有滚轮的情况
    • state_pressed:控件是否被按下
    • state_active:控件是否处于活动状态,eg:slidingTab
    • state_single:控件包含多个子控件时,确定是否只显示一个子控件
    • state_first:控件包含多个子控件时,确定第一个子控件是否处于显示状态
    • state_middle:控件包含多个子控件时,确定中间一个子控件是否处于显示状态
    • state_last:控件包含多个子控件时,确定最后一个子控件是否处于显示状态

  • Item

    <item>标签可以理解为给当前drawable赋值的属性（如textColor），因此可以添加属性支持的值，如

    <item android:color="@color/black"/>

Handler

大致流程：

线程启动，初始化 Looper，Looper内部初始化 MessageQueue，调用 Looper.loop()开启循环，MessageQueue以链表形式遍历 Message 并分发，由 Message 内存储的 Handler 或 Runnable 来消费消息

当 MessageQueue 无消息时（queue.next()会阻塞），线程进入休眠，当有新的 Message 入队后会唤醒 Looper 所在线程并重新取消息

handleMessage运行的线程与创建Handler的线程是同一线程（也可以在构造函数里传入某线程的looper，就会初始化到那个线程），可以通过这种方式切到主线程

不能在没有Looper的前提下去使用Handler（处理回调的线程必须有轮询）

  • Message：消息对象,是在线程之间传递的消息,它可以在内部携带少量信息.可以传递bundle数据

    由于 Message 采用了缓存复用机制，当 handleMessage 方法被回调后，Message 携带的所有参数都会被清空，而如果外部的 handleMessage方法是使用了异步线程来处理 Message 的话，那么异步线程只会得到一个空白的 Message

  • MessageQueue：存储消息对象的队列,每个线程只有一个,阻塞队列.

  • Looper：负责循环读取MessageQueue中的消息,读到消息之后把消息交给Handler去处理.是每个线程中MessageQueue的管家,调用Looper.loop()方法后,就会进入到一个无限循环中,会从头开始取消息并传递到Handler的handlerMessage()方法中,每个线程只有一个looper对象。其中会不断调用messageQueue的next()方法，当有消息就处理，否则阻塞在messageQueue的next()方法中

    主线程中vSync、用户操作、系统广播等都属于消息，消息到来前主线程被阻塞,会释放CPU资源,进入休眠状态，消息到来时就从休眠中唤醒，处理完新消息后，再次queue.next()进入休眠，如此反复。

    ANR并非因为消息等待阻塞引起，而是消息处理时间过长，无法及时响应下一个消息

  • Handler：发送消息与处理消息.在其子线程或当前线程发送Message,在UI线程处理消息对象,发送的消息经过处理后最终在handlerMessage方法中处理.

  • 消息屏障：消息屏障是一种特殊消息，MessageQueue遍历到该屏障消息时，就会先跳过同步消息，优先执行异步消息。我们可以通过调用 Message 对象的 setAsynchronous(boolean async) 方法来主动发送异步消息，但不能发送消息屏障消息（可以通过反射调用 MessageQueue 的 postSyncBarrier() 方法来发送），所以设置了异步消息后也不一定马上执行

内存泄漏

当退出 Activity 时，如果作为内部类的 Handler 中还保存着待处理的延时消息的话，那么就会导致内存泄漏，可以通过调用Handler.removeCallbacksAndMessages(null)来移除所有待处理的 Message
